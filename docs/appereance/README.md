# Appearance

Ten folder zbiera dokumenty dotyczące wyglądu aplikacji.
Jeśli decyzja dotyczy kolorów, czcionek, kafelków, układu ekranów, animacji, ikon, top baru, dolnego menu albo ogólnego odczucia aplikacji, zapisujemy ją tutaj.

## Pliki

### `design.md`
Roboczy design system aplikacji.
Ten plik opisuje aktualny wygląd aplikacji jako całość:
- kolory
- czcionki
- promienie
- kafelki
- przyciski
- inputy
- top bar
- dolny panel
- animacje
- ogólne zasady wizualne

Ten plik odpowiada na pytanie: "Jak wygląda aplikacja jako system wizualny?"

### `UX_UI.md`
Opis UX/UI poszczególnych ekranów i sekcji.
Ten plik opisuje, jak mają wyglądać i zachowywać się konkretne części aplikacji:
- Dashboard
- Kontakty
- Umówione spotkania
- W realizacji
- Statystyka
- Konto / Ustawienia otwierane z topbara
- popupy
- kafelki i listy
- gesty i przejścia

Ten plik odpowiada na pytanie: "Jak wygląda i zachowuje się konkretny ekran albo moduł?"

## Zasada pracy
Jeśli opis dotyczy ogólnego stylu, dopisz go do `design.md`.
Jeśli opis dotyczy konkretnego ekranu albo interakcji, dopisz go do `UX_UI.md`.
Jeśli nie wiadomo gdzie coś wpisać, wpisz tymczasowo do `UX_UI.md`, a później uporządkujemy.
